# klik — Giper Baza load test

Кликер для проверки, сколько одновременных игроков тянет одна сессия в Гипер Базе.

Prod: https://b-on-g.github.io/klik/

## Идея

- Каждый игрок = `$bog_klik_player` в общем ленде сессии (Score + Name).
- Ручной клик или автокликер инкрементит Score раз в `tick_ms`.
- Лидерборд читает всех игроков и сортирует по Score.
- Страница **Нагрузка** спавнит N фейковых ботов в одной вкладке — моделирует N одновременных писателей.

## Как пользоваться

1. Открыть `/klik/`, нажать **Создать сессию** на вкладке Клик.
2. В URL появится `?land=<link>`. Скопировать и разослать.
3. Все по этой ссылке попадают в одну сессию, кликают, видят общий лидерборд.
4. На **Нагрузка** поставить число ботов и включить — они стартуют авто-инкрементами.

## Что можно измерить

- Размер пакета апдейта на игрока (`diff_parts`/`pack` из Giper Baza).
- Ops/s на аплае пакетов на сервере.
- Лаг роундтрипа клиент → сервер → клиент.
- Зависимость от количества игроков / частоты тика.

## Структура

- `store/store.ts` — `$bog_klik_player`, `$bog_klik_store` (враппер glob + url-arg)
- `play/` — клик + автокликер для одного игрока
- `board/` — лидерборд (читает dict, сортирует по Score)
- `bench/` — спавн ботов как ключи в общем dict, метрики (ops/s, лаг)
